Hello! I am DevLens, and I am here to help you understand the **anikchand461_ragbucket** repository.

**RagBucket** is a tool designed to solve the fragmentation in RAG (Retrieval-Augmented Generation) systems by packaging your entire retrieval pipeline—including vectors, chunks, configuration, and runtime metadata—into a single, portable `.rag` artifact.

Key aspects of RagBucket include:

*   **Portability:** It treats RAG systems as self-contained intelligence artifacts that can be built once and deployed anywhere, eliminating the need for external infrastructure like dedicated vector databases.
*   **Decoupling:** It separates retrieval memory (what you built) from language generation (how you query it), allowing you to swap LLM providers at runtime without rebuilding the pipeline.
*   **Technology Stack:** It utilizes FAISS for vector search, Sentence Transformers for embeddings, LangChain for chunking, and Pydantic for configuration validation.
*   **Ease of Use:** It provides a simple Python interface to load an artifact and query it, with support for multiple LLM providers like Groq, OpenAI, Gemini, and Anthropic.

Hello! I am DevLens, and I am here to help you understand the **anikchand461_ragbucket** repository.

The core features of RagBucket include:

*   **Portable Artifacts (.rag files):** Packages your entire RAG pipeline—vectors, chunks, configuration, and metadata—into a single, self-contained, and shareable file.
*   **Infrastructure Decoupling:** Separates retrieval memory (what you built) from language generation (how you query it), allowing you to swap LLM providers at runtime without rebuilding.
*   **Reproducibility:** Ensures 100% reproducible retrieval results by baking the chunking configuration and indexing into the artifact.
*   **Flexibility:** Supports multiple LLM providers (Groq, OpenAI, Gemini, Anthropic) and various embedding models (OpenAI, Cohere, Gemini, Voyage AI, and local Sentence Transformers).
*   **Built-in Vector Search:** Uses FAISS (Facebook AI Similarity Search) for semantic indexing with zero-config setup.
*   **Self-Describing:** The artifact contains everything needed to query it, eliminating the need for external infrastructure dependencies or scattered configuration files.
